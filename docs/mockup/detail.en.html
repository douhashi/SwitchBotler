Air Conditioner controls — slider-first, refined

A large temperature readout plus a bold Soft Depth slider. The accent color follows the operating mode (Cool = blue / Heat = orange / Dry = teal). Fine-tune with −/＋, and pick mode and fan speed by icon. Slider-driven, with no circular dial.

Implementation notes (proposal)
· Data stays as-is (temperature 16–30 / mode auto·cool·dry·fan·heat / fanSpeed auto·low·medium·high / power). Only the UI is refreshed; the API send logic is unchanged.
· Temperature uses the existing Slider, made thicker and filled with the mode color. −/＋ calls updateControl({ temperature: clamp(t±1,16,30) }).
· A single mode → accent color map links the fill, the large number, the mode chip, and the selected mode (icon + label always accompany the color to ensure accessibility).
· In Fan mode, temperature is disabled (many models don't allow setting a temperature). One option is to show the slider in a disabled state.
· The screen is device-detail (device control). If the direction is approved, split it out as docs/mockup/detail.html (+en) from index's #02.
