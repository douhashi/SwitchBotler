Effort: Medium Variable height + max

Add an upper bound to the content-fitting window height (current ResizeObserver) and clamp to it. Within the bound it flexes as before.

Effort: Medium Independent scroll

Give the device region and scene region their own max-height + overflow-y. Scenes never get buried below devices and stay reachable.

Effort: Low Bottom fade

Overlay a fade at the bottom of each scroll region to signal "there's more" (a cue that doesn't rely on color alone).

Implementation notes (proposal)
· Full control: Remove the hasPowerToggle filter in tray-popover.tsx and branch via deviceInteraction() into toggle=switch / detail=">" / press="Press" (sharing the exact logic of the device cards).
· Where ">" leads: Show the main window and jump to that device's detail. Today show_main_window(view) can only pass a view, so extend the IPC to pass a device id (attach the id to the navigate event). This is the only non-trivial point (Effort: Medium).
· Width: Set the tray window width to about 288→360px. Update the width in tray-app.tsx's setSize (switch the window.innerWidth dependency to a fixed width).
· Variable height + cap: Add Math.min(h, MAX) to the current behavior that follows the content height via ResizeObserver. Give each of the device/scene lists max-height + overflow-y: auto (devices ≈ 6 rows / scenes ≈ 3 rows).
· UX note: Nested scrolling inside a small popover needs care (use overscroll-behavior: contain to stop propagation to the parent). A "single-scroll" variant limited to just one region is also possible (pick per preference).
