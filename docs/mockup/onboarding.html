Design Mockup · 未接続時の体験

オンボーディング画面とトレイの表記

Token / Secret が未設定（saved:false）のとき、サイドバー付きの通常シェルではなく、接続だけに集中したメニューなしのオンボーディング画面を表示します。あわせてトレイポップアップの「未接続」表記を、状態に応じて設計し直します。

01

メニューなしオンボーディング画面

推奨 未設定時のメインウィンドウ全体。1画面で「知る→取得→接続」を完結。

SwitchBotler へようこそ

お使いの SwitchBot デバイスを、メニューバーからワンタップで操作できます。はじめに API 認証を設定しましょう。

1

アプリで発行

SwitchBot アプリ → プロフィール → 設定 → 開発者向けオプション

2

2つの値をコピー

トークンとシークレットをそれぞれコピーします。

3

貼り付けて接続

下のフォームに貼り付け、「保存して接続」。

入力した値は OS のセキュアストレージ（Keychain / Credential Manager / Secret Service）に保管され、画面やログに平文で表示されません。
開発者向けオプションの開き方 →

メニューを出さない理由

接続前はデバイス・シーン・センサーの各画面が空で、操作しても意味がありません。ナビを隠すことで次の一手（＝接続）だけに視線を集約します。

1画面で完結

ウィザードにせず1画面に。手順はstepストリップで示しつつ、入力・保存まで同一視野で行えます（YAGNI）。

接続後の遷移

接続成功で saved:true となり、通常のサイドバー付きシェル（デバイス一覧）へ自動遷移。以降このオンボーディングは出しません。

表記の対応表

状態を3つに分け、それぞれ命名・本文・アクションを変える。

未設定 !saved

バッジ: セットアップが必要
見出し: 接続をはじめましょう
本文: SwitchBot の API 認証がまだ設定されていません。
操作: セットアップを開く（→ オンボーディング）

到達不能 saved & error

バッジ: オフライン
見出し: 接続できませんでした
本文: SwitchBot API に到達できません。ネットワークまたは認証をご確認ください。
操作: 再試行

接続済み connected

バッジ: 接続済み
本文: お気に入りのデバイス／シーンを表示。
旧「未接続」を全状態に流用 → 状態ごとに命名を分離。

実装メモ（提案）
・ 分岐キーは接続ステータスではなく connection.saved。!saved で <AppShell> の代わりに <Onboarding> を描画（サイドバー・ナビを出さない）。
・ saved && disconnected（到達不能）は通常シェルを維持し、設定画面で再テスト。トレイは「オフライン／再試行」。
・ トレイの showDisconnected を 未設定 と 到達不能 の2分岐へ（現状は一律「未接続です。設定から接続してください。」）。
・ トークンのみ発行 → シークレット未入力のような中間状態は作らず、両方揃って初めて saved とする（現行の saveCredentials と一致）。
